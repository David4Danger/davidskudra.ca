Riot API Challenge 2.0 - Project Documentation Part 4

Posted on 8/28/2015

As of today, I have completed the website that is displaying all the data I've gathered for my entry into the Riot API Challenge 2.0. You can find this wesbite at www.abilitypowerparity.com. In this post, I'll be giving a quick explanation on how to navigate the website and what to look for.

The first thing you'll notice is the homepage. Here you can choose any of the twelve AP items displayed to begin. Let's start with Rod of Ages, for an example. Once you click the Rod of Ages icon, you're taken to the webpage that hosts all the information on Rod of Ages. On the leftmost tab, there are two small columns that display the build path of the item, and all the stats provided that changed. For example, under the patch 5.11 column you'll see a different price for Blasting Wand, a different amount of additional gold needed to finish Rod of Ages, and four different statistical changes made to the item itself. Under these changes at the bottom lies a single column which details all the stats of the item that remained the same after it was updated. In some cases the stats don't change it all, in others the stats are the only thing that changed.

This method of display makes comparing the item before and after it was updated incredibly easy, providing a great base for understanding the middle column.

The middle column describes the results of the changes made to each item - here you'll find winrates, popularity, and gamelength information when you first load up the page. This is a good reference point for each item, creating an idea of how the changes affected the item in different aspects. This data isn't the only thing that is displayed in the second column, which in turn leads to the rightmost column.

What you'll see on the right is an array of buttons, each proposing a different set of information. Clicking one of the buttons will hide the base dataset in the middle column, and display a different one according to which button is selected. For example, clicking the "Anivia Building Rod of Ages" button will display some information that is unique to the item and champion.

One button that you'll see on every page in the rightmost column is the "Item Summary" button. Selecting it will display some information that helps paint a general picture of how these changes affected each item, and the implications of these changes. This is the quickest way to get a good idea of what happened to each item.

The last thing to cover here as that at the top of each page, you'll find twelve different icons to interact with. The first eleven on each page will take you to the pages of each respective AP item (other than the one you're currently on). The rightmost icon will bring you back to the homepage as well.

One thing you might notice while exploring the website is that some of the pages have more buttons to choose from on the rightmost tab than other pages. For the most part, this is proportional to the severity of the changes for each item, most importantly it's popularity. Luden's Echo had the most tabs because of how popular it was and how large the changes made to it were.


All in all, this website has the goal of getting the information I acquired to the reader as clear as possible. Each page starts by detailing what was changed, then how it affected basic stats like winrate, popularity, and game length, then finishing with some extra information to help expand knowledge on the topic once a solid foundation has been created. The intention of the provided layout is to almost read like a book, starting with the leftmost item, and then reading the information on each page like the page of a book; left to right. Of course a lot more people (as shown by popularity) care about Luden's Echo, compared to something like Nashor's Tooth, so there might be some skipping of "pages".

You can find a full set of documentation on how the data acquisition files I created worked and how I interfaced with them to get all my information at my github page for the API Challenge entry. What you'll find there will be much more technical.

Thanks for reading my progress logs as I created my project for the Riot API Challenge 2.0, and I hope you'll explore my webpage further to see more!
