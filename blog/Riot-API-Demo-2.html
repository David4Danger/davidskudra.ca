File 1 - RiotConstants.py

The first file we'll be looking at is the file called RiotConstants.py. This file will be holding all the constants that we use to execute a request with the Riot API. The file has three constants that each contain a dictionary; the first dictionary is the most important as it is the unformatted url we make requests with. The following two dictionaries are made up of keys and values that are used to help format the first dictionary. Take note of the comments in the code below, and then I'll be going into more depth about the code itself once you've seen it.



As described above, this file has three constants that are composed of dictionaries. First off, we can see that the 'URL' dictionary is composed of 3 keys: base, summoner_by_name, and stats_by_summoner summary. The first of the three represents the base URL that is used in most requests sent to the Riot API. Take note that it has three variables inside the string that we are formatting: proxy, region, and url. The first two variables are going to be formatted to be one of the values in the 'REGIONS' dictionary. The last variable, url, is formatted to be one of those two remaining values in the URL dictionary. This will all be done in the next file we work on.

This takes us into the next two keys of URL. The second key, summoner_by_name, gives us another value that requires formatting. This string will be formatted with 2 variables: version and names. Version will be formatted with a value from the next dictionary, 'API_VERSIONS', while names will be a summoner name we enter in the next file we look at, on page 3.

The second constant in RiotConstants, 'API_VERSIONS', is a simple one. As found on the Riot API full reference, each request has a current API version. It isn't updated too often so expect that once you fill this dictionary in, you won't have to change the value of any keys you've already put in this dictionary. As mentioned before, 'API_VERSIONS' is used to format the previous dictionary we described, 'URL'.

The final constant in RiotConstants is 'REGIONS'. We mentioned this constant before as it is used to format the value of base in the URL dictionary. This constant is the most static of the three, as once you enter information you will never need to update it. This dictionary is composed of the regions you want to use, and their matching acronyms. This information can all be found on the Riot API full reference, but if you're only looking to use the main 3 regions you are fine just copying my code from here.

As a short footnote, earlier I pointed out that the base url in the URL dictionary is used in most requests, but not all of them. There are two requests you'll find on the full api reference called "current-game" and "featured-games", those use a different base URL that I'm not covering in this demo. If you understand what I've done on this page and want to utilize those requests, coming up with the proper "base2" constant is pretty easy, consider it an exercise.


So we've now defined all the constants we need to work with in our application and have a good understanding of how they work. A quick refresher before we move on to the second file:

  • The 'URL' dictionary is fairly static and doesn't require much upkeep. There is almost 100 API requests you can make and I've only included 2, so check out the full reference and add some more requests!
  • The 'API_VERSIONS' dictionary requires a bit of maitenance just to make sure you're using the latest version of each API. Just keep an eye on the versions being used on the full reference, and update the values as needed.
  • The 'REGIONS' dictionary is as static as it gets. There is about a dozen different regions you can put in this dictionary, but which you put in there us up to you.
  • If you end up making a web application, make sure you include all the possible regions in the 'REGIONS' dictionary. You could have your largest target audience in the NA region, but it is easy and thoughtful to include all the possible regions.
