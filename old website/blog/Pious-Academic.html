16807 Pious Academic

Posted on 12/27/2015

I'm not quite sure how I actually found the time to make this library in the midst of my recent study term, but my motivation was the result of both a passion for Halo and how excited I was to be able to create a public library to share with others. After my recent experience using the Riot Games API, I decided to direct my talents at writing an unofficial python library to use the 343 Industries API created for Halo 5. The following is the result of my efforts!

You might be wondering about the name. I won't bore you with Halo related lore but my inspiration for the name for the name was from the forerunner monitors you see often in the series. That, and since 343 is 7 raised to the power of 3, I took it a step further and made it 7 raised to the power of 4.

Before I get into the code, here is a link to the public library I've been referencing. The first thing you might notice is that this library uses one of my favourite python libaries, requests. This library makes processing HTTP requests much simpler in python. As abstraction goes, you need not understand the underlying processes behind requests but what its functions actually do.

The code itself is heavily commented so I'll just be highlighting a few key concepts that help make this library work. The first of which is that I included a class that handles rate limiting on a local scale. As with most public APIs, users have a limit that constrains how often you can make requests to the API. One thing I noticed in a few various other libraries made for this API is that local rate limiting wasn't considered. My ratelimit class protects both the user and the host of the API as you cannot exceed your specific rate limit when executing requests.

In the request class, there are 3 specific request functions that get reused for all the various get requests. Each is similar in nature but has slightly different URL formatting. The individual get requests are already equipped to use the correct request function but it is important to understand that they must use the proper request in order to format a proper endpoint.

Outside of the main file I also wrote getmeta.py, which is used to create a local copy of all the metadata currently available from the API. It creates a local JSON for each meta request to save the user the trouble of executing the same meta request multiple times. All you have to do it format the file pathing and run getmeta once in a while to make sure your metadata is up to date.

As of now I've yet to personally create a project using my API but I have intentions to utilize in collaboration with bigteambattle.net for an upcoming project, so stay posted for that!
